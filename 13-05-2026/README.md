Nesta análise whitebox do challenge Weather App (Hack The Box), comparamos a eficácia do Semgrep e do Gemini 3.1 na detecção de vulnerabilidades em código fonte. O experimento mostrou a diferença técnica entre ferramentas SAST e a versatilidade de LLMs aplicados a CTFs. Enquanto o Semgrep detectou misconfigurations válidas, porém periféricas, como contêineres operando como root, chaves de API fixas no código e ausência de middlewares CSRF, ele não conseguiu rastrear o fluxo lógico do ataque. Em contrapartida, o Gemini 3 realizou um teste mais contextual (com ajuda de um prompt detalhado) e mapeou a exploit chain exata necessária para o comprometimento total do alvo. A análise revelou que o abuso do parâmetro endpoint no WeatherHelper.js permite um SSRF que, combinado com CRLF Injection, resulta em HTTP Request Splitting. Esse comportamento é utilizado para forjar uma requisição POST interna partindo de 127.0.0.1, contornando com sucesso o filtro de IP estabelecido para proteger a rota de registro. Através dessa requisição interna forjada, é entregue um payload de SQL Injection que explora queries não parametrizadas no database.js, utilizando a cláusula ON CONFLICT(username) DO UPDATE do SQLite para sobrescrever a senha do administrador. O ataque culmina no Account Takeover do usuário admin, permitindo o acesso à rota restrita de login e a consequente extração do conteúdo de /app/flag. Confira o artigo completo na pentestwithai para entender em detalhes como utilizar a IA não apenas para identificar falhas isoladas, mas para conectar pontos cegos na arquitetura e construir vetores de ataque complexos em suas revisões de código.
