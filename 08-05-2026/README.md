Mesma máquina. Mesmo modelo. Mesmo agente. A única diferença foi um arquivo de instruções.
A máquina Shocker do HackTheBox foi comprometida duas vezes com Claude Code + Opus 4.7. Na primeira vez, o agente recebeu um SKILL.md com metodologia PTES, mapeamento MITRE ATT&CK e regras de reporting. Na segunda, recebeu só o IP.
Com o SKILL.md, o agente resolveu em 4 minutos e 50 chamadas API. Identificou Shellshock de primeira, exploitou com um curl limpo, encontrou sudo perl NOPASSWD via sudo -l, escalou com uma linha de Perl, e gerou um relatório completo com cadeia de ataque mapeada no MITRE ATT&CK.
Sem o SKILL.md, levou quase 7 minutos. O agente também achou Shellshock, mas precisou de várias tentativas até acertar o formato do curl. Na escalada, ignorou o sudo perl (caminho simples) e foi pro LXD group abuse: construiu imagem Alpine localmente, serviu via HTTP, importou no LXC, criou container privilegiado e montou o filesystem do host. Funcionou, mas deu volta no quarteirão inteiro pra chegar no mesmo lugar. Abriu reverse shell e servidor HTTP em background sem necessidade. E no fim, gerou só um flags.txt com as respostas, sem relatório estruturado.
A diferença não foi de capacidade. O modelo sabia resolver dos dois jeitos. A diferença foi de direção. O SKILL.md fez o agente escolher o caminho mais eficiente, manter foco, e entregar um output profissional. Sem ele, o agente explorou mais, errou mais, e entregou menos.
Pra quem usa IA em pentest, o investimento não é só no modelo. É no prompt que direciona ele. Um bom SKILL.md transforma um agente capaz num agente mais eficiente.
